# ドキュメントの原則

ドキュメントに少しでも誤った情報があると、ドキュメント全体の信頼性が損なわれる。

## 変更時の影響範囲確認

ソースコードを修正・変更・追加・削除した場合、その影響範囲をドキュメントを含めて確認し、影響を受ける全てのドキュメントを更新すること。

確認対象:
- plan.md（作業ステップのチェックリスト、ディレクトリ構成、技術スタック）
- README.md（使い方、対応する変換、例）
- CLAUDE.md（アーキテクチャ、コマンド、規約）
- コードのドキュメントコメント（`///`, `//!`）

## タイミング

ドキュメント更新はコード変更と同じタイミングで行う。後回しにしない。
