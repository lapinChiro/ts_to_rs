# ドキュメント更新

## トリガー

ソースコードを修正・追加・削除したとき。

## アクション

以下のドキュメントについて、今回の変更で内容が不正確になっていないかを確認し、不正確な箇所を更新する:

- `plan.md`（作業ステップのチェックリスト）
- `README.md`（使い方、対応する変換、例）
- `CLAUDE.md`（アーキテクチャ、コマンド、規約）
- コードのドキュメントコメント（`///`, `//!`）

ドキュメント更新はコード変更と同じタイミングで行う。

## 禁止事項

- ドキュメント更新を「後でやる」として先送りすること
- コードを変更したのにドキュメントの確認をスキップすること
- 変更していない機能のドキュメントまで書き換えること（スコープ外の改善は別タスク）

## 検証

- 変更した機能に関するドキュメントの記述が、変更後のコードの挙動と一致している
