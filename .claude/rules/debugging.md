# デバッグ・エラー修正の原則

## トリガー

修正を試みて 1 回で成功しなかったとき。

## アクション

次の修正を試みる前に立ち止まり、以下の手順を踏む:

1. **情報収集**: バージョン、依存関係、環境情報など、関連する事実を集める
2. **仮説構築**: 集めた情報から根本原因の仮説を立てる
3. **仮説検証**: 仮説を検証するコマンドを実行する
4. **修正**: 根本原因が特定できてから修正する

## 禁止事項

- エラーメッセージの表面的な指示に従って修正を繰り返すこと（「missing import」→ import 追加、を根本原因を考えずに繰り返す等）
- 同じ修正を 2 回以上試みること
- 根本原因の仮説を立てずに修正コードを書くこと

## 検証

- 修正が 2 回目以降の場合、仮説を明示的に言語化してからコードを変更している
