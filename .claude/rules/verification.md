# 検証の原則

検証は「実行して結果を見る」行為ではなく、「計画 → 実行 → 判定」のプロセスである。
この原則は手動確認・自動テスト設計・品質チェックのすべてに適用される。

## プロセス

1. **検証項目の列挙**: 何を検証するかを事前にMECEに洗い出す
2. **期待結果の定義**: 各項目について、どのような結果なら合格/不合格かを実行前に決める
3. **実行**: 検証を実行する
4. **判定**: 事前に決めた基準に照らして結果を評価する

## アンチパターン

- 実行して出力を「見る」だけで確認完了とする
- 正常系だけ確認して異常系を確認しない
- 結果が出てから「これは期待通りだ」と後付けで判断する
