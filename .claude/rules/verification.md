# 検証の原則

## 適用条件

何かの正しさを確認する場面すべて（テスト設計、手動確認、品質チェック、動作検証）。

## 制約

検証は「計画 → 実行 → 判定」のプロセスである:

1. **検証項目の列挙**: 何を検証するかを事前に MECE に洗い出す
2. **期待結果の定義**: 各項目について、合格/不合格の基準を実行前に決める
3. **実行**: 検証を実行する
4. **判定**: 事前に決めた基準に照らして結果を評価する

## 禁止事項

- 実行して出力を「見る」だけで確認完了とすること
- 正常系だけ確認して異常系を確認しないこと
- 結果が出てから「これは期待通りだ」と後付けで判断すること
- 期待結果を定義せずに検証を開始すること
