## 3. 修正済み: テスト不足だった構文

### 3.1 Update式 (`i++`, `i--`) — 修正済み (I-108)

`convert_expr()` に `ast::Expr::Update` 分岐を追加。forループ専用だった変換ロジックを式レベルで汎用化。
- ユニットテスト追加: postfix increment/decrement
- スナップショットテスト追加: `update-expr.input.ts`
- E2Eテスト追加: `update_expr.ts`

### 3.2 Arrow関数のデストラクチャリングパラメータ — 修正済み (I-109)

Arrow関数のパラメータ処理に `Pat::Object` と `Pat::Assign` を追加。`functions/mod.rs` の既存ロジック（`convert_object_destructuring_param`, `convert_default_value`）を再利用。
- ユニットテスト追加: 型注釈付きデストラクチャリング、デフォルト値パラメータ

### 3.3 クラスのデフォルト値パラメータ — 修正済み (I-110)

`classes.rs` の `convert_param_pat` に `Pat::Assign` を追加。コンストラクタ/メソッドの本体先頭にデフォルト値展開ステートメントを挿入。
- ユニットテスト追加: 数値デフォルト値、空オブジェクトデフォルト値
- スナップショットテスト追加: `class-default-params.input.ts`

## 4. 新たに表面化した未対応構文

修正により、隠れていた次のエラーが3件表面化:

1. **非リテラルデフォルト値** (mime.ts, concurrent.ts): `mimes: Record<string, string> = baseMimes` — デフォルト値が変数参照。`convert_default_value` がリテラルと空オブジェクトのみ対応のため。既知の制限 (TODO スコープ外: デフォルト引数値の拡張)
2. **ビット演算子 `^`** (buffer.ts): `va.getUint8(i) ^ vb.getUint8(i)` — XOR演算子が未対応。`convert_bin_expr` の対応演算子に追加が必要
3. **Arrow param pattern の残り** (middleware/basic-auth/index.ts): 型注釈なしのデストラクチャリング `[usernameEqual, passwordEqual]` — 配列デストラクチャリングは Arrow param で未対応

## 5. 今後の優先度提案

### 最優先（影響ファイル数最多）

1. **オブジェクトリテラルの型推論** (I-112): 10ファイルに影響。変換率改善の最大のボトルネック
2. **正規表現リテラル** (I-111): 8ファイルに影響

### 次優先

3. **型エイリアスのスキップ戦略**: 8ファイル。ランタイムコードには影響しないが、型宣言でエラーになり後続の変換が止まる。スキップして継続する戦略が有効
4. **ビット演算子 (`^`, `|`, `&`, `~`)**: 追加コストは小さい

5. 非リテラルデフォルト値、for...in、関数式、etc.
