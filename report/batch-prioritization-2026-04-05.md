## Step 2: レベル判定

### L1: 信頼性基盤

**判定基準**: コンパイル成功 + ランタイム挙動がTSと異なる / テスト基盤の信頼性毀損

| RC | 判定 | 根拠 |
|----|------|------|
| S1 (I-298) | **L1** | spread 上書きで副作用式の評価が消失。コンパイル成功するが TS と異なるランタイム挙動。S1 の定義に直接該当 |

### L2: 設計基盤

**判定基準**: 今後の開発のたびに同一クラスの問題が再発する設計的欠陥

| RC | 判定 | 根拠 |
|----|------|------|
| RC-15 (I-299+I-273) | **L2** | convert_ts_type が呼ばれる全箇所（型変換の中核パス）で型パラメータ制約が欠如。新しい型変換を追加するたびに同一制約に直面する。責務違反 |
| RC-14 (I-282) | **L2** | `build_option_unwrap_with_default` の needs_lazy 判定が不完全。副作用式のデフォルト値が追加されるたびに同一バグが再発する設計。DRY 統合で修正箇所 1 箇所に集約済みのためコスト小 |

### L3: 拡大する技術的負債

**判定基準**: 放置するとコストが増大 / 他イシューのブロッカー / ゲートイシュー

| RC | 判定 | 根拠 |
|----|------|------|
| RC-11b (I-300+I-301+I-306) | **L3** | OBJECT_LITERAL_NO_TYPE 25件。ベンチマーク最大エラーカテゴリ。レバレッジ最大 |
| RC-9 (I-311+I-344) | **L3** | I-311 解決で I-344 自動解消。generic 関数増に比例して拡大 |
| RC-11a (I-361) | **L3** | デストラクチャ変数の後続型推論が全滅。拡大中 |
| RC-13 (I-238+I-202+I-11) | **L3** | 新 union 生成のたびに同一パターンの問題が発生。拡大中 |
| RC-2a (I-340) | **L3** | generic コード増に比例して拡大 |
| RC-2b (I-217+I-265) | **L3** | iterator パターン増に比例。skip 原因 (array-builtin-methods) |
| RC-5 (I-336+I-337) | **L3** | abstract class 使用箇所が限定的で安定。拡大速度は低い |
| RC-20 (I-360+I-331) | **L3** | Option<T> 使用増に比例。I-360 は fixture `functions` のスキップ原因 |

| GATE (I-30) | **L3** | I-183, I-34 のゲート |
| CI (I-182) | **L3** | 回帰検出不能は品質リスク |
| RC-38 (I-313) | **L3** | CALL_TARGET 4件。変換エラーの直接原因 |
