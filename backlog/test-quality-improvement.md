# テスト品質の改善

## 背景・動機

正当性チェックで以下のテスト品質問題が確認された:

1. **7 個のスナップショットがコンパイル不可のコードを期待値に持つ**（compile_test でスキップ）
2. **12 個の重要シナリオにテストが存在しない**
3. **statement テストで `matches!()` による弱いアサーション**が多用
4. **throw の Result ラッピングテストが不十分**（返り値型のみチェック、本体未検証）

## ゴール

- コンパイル不可スナップショットを修正するか、不可の理由をテスト内にドキュメント化する
- Critical/High の欠落テストを全て追加する
- 弱いアサーションを強化する

## スコープ

### 対象

- スナップショット修正（`builtin-api-batch` の 7 箇所のコンパイルエラー）
- 欠落テストの追加（12 シナリオのうち、他 PRD で対応されないもの）
- `matches!()` アサーションの `assert_eq!()` への強化（主要テスト）

### 対象外

- 他 PRD で対応される機能の修正（テストのみ追加、実装は別 PRD）
- 全テストの網羅的書き直し

## 設計

### 技術的アプローチ

**スナップショット修正:**

`builtin-api-batch` の具体的な問題:
- `findIndex` → `position()` は `Option<usize>` を返す → `.map(|i| i as f64).unwrap_or(-1.0)` に修正
- `sort_by` のクロージャ → `a.partial_cmp(b).unwrap()` パターンに修正
- `drain` の範囲式 → 括弧で優先順位を修正

**欠落テストの追加（他 PRD と重複しないもの）:**

| # | シナリオ | テスト内容 |
|---|---------|----------|
| 1 | `const` オブジェクトのフィールド変更 | `const obj = {x:1}; obj.x = 2` の変換結果確認 |
| 2 | ネスト optional chaining | `x?.y?.z` の `Option` 型確認 |
| 3 | 三項演算子の異型分岐 | `cond ? "a" : 1` の変換結果確認 |
| 4 | arrow 関数のオブジェクト分割代入 | `({a, b}) => a + b` のエラー確認 |
| 5 | `Math.max(a, b, c)` 3 引数 | 現在の変換結果と期待の乖離を確認 |

**アサーション強化:**

statement テストの主要なもの（try/catch、for-loop、do-while）で `matches!()` → `assert_eq!()` に変更。

### 影響範囲

- `tests/snapshots/` — スナップショット更新

- `src/transformer/*/tests.rs` — テスト追加・アサーション強化
