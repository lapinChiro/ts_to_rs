# spread の非先頭位置対応

## 背景・動機

現在のオブジェクト spread 変換（`{...p, x: 10}` → `Point { x: 10.0, y: p.y }`）は spread が先頭にある場合のみ対応している。Hono では `{ key: value, ...rest }` のように spread が先頭以外の位置にあるパターンが存在し、変換エラーになる。

## ゴール

`{ x: 10, ...rest }` が正しく変換される。具体的には、明示的に指定されたフィールドを優先し、残りを spread 元から取得する Rust コードが生成される。

型注記付きの場合: `const p: Point = { x: 10, ...rest }` → `Point { x: 10.0, y: rest.y }`（spread 元のフィールドで補完）

## スコープ

### 対象

- spread がオブジェクトリテラルの先頭以外にある場合の変換
- spread が途中にある場合（`{ a: 1, ...rest, b: 2 }`）の変換
- 型注記があるケースのみ（struct 名が判明しているケース）

### 対象外

- 型注記なしのオブジェクト spread（struct 名が不明のため、そもそもオブジェクトリテラル変換の対象外）
- 配列 spread の非先頭位置（別の問題）

## 設計

### 技術的アプローチ

既存の spread 変換ロジック（`expressions.rs` のオブジェクトリテラル処理部分）を拡張する:

1. オブジェクトリテラル内の全プロパティを走査
2. spread 以外の明示的プロパティのフィールド名を収集
3. TypeRegistry から struct の全フィールド名を取得
4. 明示的に指定されていないフィールドを spread 元から補完
5. spread の位置に関わらず、同じ出力を生成する

現在の実装では spread が先頭にある前提でハードコードされている部分を、位置に依存しないロジックに一般化する。

### 影響範囲

- `src/transformer/expressions/mod.rs` — オブジェクトリテラル変換の spread 処理部分
