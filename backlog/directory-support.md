# ディレクトリ対応

## 背景・動機

現在の CLI は単一ファイルの変換のみ対応している（`ts-to-rs input.ts -o output.rs`）。
実際の TS プロジェクトは複数ファイルで構成されるため、ディレクトリを指定して一括変換できる必要がある。

## ゴール

- `ts-to-rs src/` のようにディレクトリを指定すると、配下の `.ts` ファイルを再帰的に変換する
- ディレクトリ構造を維持したまま `.rs` ファイルを出力する
- Rust のモジュール構造に必要な `mod.rs` を自動生成する

## スコープ

### 対象

- ディレクトリ入力時の再帰的な `.ts` ファイル探索
- 出力先ディレクトリの指定（`-o` オプション）
- ディレクトリ構造のミラーリング（`src/foo/bar.ts` → `out/foo/bar.rs`）
- `mod.rs` の自動生成（各ディレクトリに `mod` 宣言を集約）

### 対象外

- `node_modules/` や隠しディレクトリの除外（固定ルールで十分）
- `.d.ts` ファイルの処理
- ファイル間の依存関係解決（import/export 対応の PRD で扱う）
- Watch モード
- Cargo.toml の自動生成（別 PRD）

## 設計

### 技術的アプローチ

1. **CLI の拡張**: `input` 引数がファイルかディレクトリかを判定
   - ファイル: 既存の動作（単一ファイル変換）
   - ディレクトリ: 再帰的に `.ts` ファイルを収集して各ファイルを変換

2. **ファイル収集**: `std::fs` の再帰走査で `.ts` ファイルを収集
   - `node_modules/`, `.` で始まるディレクトリを除外
   - `.d.ts` ファイルを除外

3. **出力パス計算**: 入力パスの相対関係を維持
   - 入力: `src/foo/bar.ts`, 出力先: `out/` → `out/foo/bar.rs`
   - 出力先ディレクトリが存在しなければ作成

4. **mod.rs 生成**: 各ディレクトリに配置
   - そのディレクトリ直下の `.rs` ファイル → `pub mod <name>;`
   - サブディレクトリ → `pub mod <dir_name>;`

### 影響範囲

| ファイル | 変更内容 |
|----------|----------|
| `src/main.rs` | ディレクトリ判定、ファイル収集、一括変換ループ |
| `src/lib.rs` | 変更なし（`transpile()` は単一ファイル単位のまま） |
| `tests/` | ディレクトリ入力のテスト追加 |

## 作業ステップ

- [ ] Step 1: `.ts` ファイルの再帰収集関数を実装（除外ルール込み）
- [ ] Step 2: 出力パス計算ロジックを実装
- [ ] Step 3: CLI のディレクトリ判定を追加し、ファイル/ディレクトリで分岐
