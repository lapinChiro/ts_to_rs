# テストカバレッジの計測と最低ライン強制

## 背景・動機

CI（GitHub Actions）は導入済みだが、テストカバレッジの計測・可視化は行っていない。変換機能が増えるにつれ、テストの抜け漏れに気づきにくくなる。最低ラインを設定して CI で強制することで、カバレッジの低下を早期に検知する。

## ゴール

- ローカルで `cargo llvm-cov` を実行し、カバレッジレポートを確認できる
- CI でカバレッジを計測し、閾値を下回った場合にビルドが失敗する
- 閾値は現在の実測値に基づいて設定する（現実的な最低ライン）

## スコープ

### 対象

- `cargo-llvm-cov` の導入と CI 統合
- ライン（行）カバレッジの閾値設定
- ローカル実行コマンドの整備（CLAUDE.md / README.md への記載）

### 対象外

- Codecov 等の外部サービス連携（将来検討）
- ブランチカバレッジ・リージョンカバレッジの閾値設定（ラインカバレッジのみで開始）
- カバレッジを上げるためのテスト追加（別タスク）

## 設計

### 技術的アプローチ

1. **ツール**: `cargo-llvm-cov`（rustc の `-C instrument-coverage` ベース、クロスプラットフォーム対応）
2. **閾値決定**: まず現在のカバレッジを計測し、その値を基準に閾値を設定する
3. **CI 統合**: GitHub Actions に `cargo llvm-cov --fail-under-lines <閾値>` ステップを追加
4. **ローカル実行**: `cargo llvm-cov` でテキストレポート、`cargo llvm-cov --html` で HTML レポート

### 影響範囲

| ファイル | 変更内容 |
|----------|----------|
| `.github/workflows/ci.yml` | カバレッジ計測ステップの追加 |
| `CLAUDE.md` | カバレッジコマンドの追記 |
| `README.md` | 開発セクションにカバレッジコマンドを追記 |

## 作業ステップ

- [ ] Step 1: ローカルに `cargo-llvm-cov` をインストールし、現在のカバレッジを計測する
- [ ] Step 2: 計測結果に基づいて閾値を決定する（ユーザーと合意）
- [ ] Step 3: CI に `cargo llvm-cov --fail-under-lines <閾値>` ステップを追加する
- [ ] Step 4: CLAUDE.md と README.md にカバレッジ関連コマンドを追記する
- [ ] Step 5: CI が正しく動作することを確認する（ローカルで同じコマンドを実行）

## テスト計画

- ローカルで `cargo llvm-cov --fail-under-lines <閾値>` が成功することを確認
- ローカルで `cargo llvm-cov --fail-under-lines 100` が失敗することを確認（閾値超え検証）
- CI ワークフローの YAML が妥当であることを目視確認

## 完了条件

- `cargo llvm-cov` がローカルで実行可能
- CI でカバレッジ計測が実行され、閾値を下回ると失敗する
- CLAUDE.md と README.md にコマンドが記載されている
